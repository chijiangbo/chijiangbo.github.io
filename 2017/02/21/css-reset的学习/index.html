css reset的学习

发表于 2017-02-21 | 分类于 css | | 阅读次数 |

不了解这个之前，总是被p body的默认样式折磨的死去活来，今天想记录一下，在以后的制作过程中，自己怎么使用reset.css以及如何使用好reset.css

reset的作用

对于body，p，等等标签会有默认的样式，，比如body的默认样式 {margin: 8px;}, p元素样式有margin-top,margin-bottom。这样在页面布局过程中，会出现好多莫名奇妙的小尴尬，所以，在布局之前，应对有些标签进行reset，以便后续的css能更快的书写。

先放结果，精简的reset.css代码如下

body, dl, dd, h1, h2, h3, h4, h5, h6, p, form{margin:0;}

ol,ul{margin:0; padding:0;}

接下来分析一下网上那些陈长的reset.css代码，从中可以学到很多知识,网上一般的reset.css代码如下

body, div, dl, dt, dd, ul, ol, li, h1, h2, h3, h4, h5, h6, pre, code, form, fieldset, legend, input, button, textarea, p, blockquote, th, td { margin: 0;padding: 0; }

抱着宁可错杀一千，也不放过一个的心态，一般的reset.css就给各种标签都设置了 { margin: 0;padding: 0; }. 那么我们来分析一下：

  • div，li标签默认没有margin,padding值，所以给它重置就是没有道理的，多此一举。
  • dt标签的默认margin,padding的值就是0，为什么还要设置为0呢？
  • code属于inline水平元素，是没有margin,padding属性的

所以，这个一遍reset.css会显得很懒，很邋遢，不精简。通过概念去判断后，就会得到我们起初给的结果，精简版的reset.css。

那么一定非要reset.css吗

答案是不一定，也应尽可能的不用它，我们在用某个标签的时候，直接设置就好了。
比如，我们现在要用到 p 标签，那么我们就可以在index.css写的时候，给p标签加上

p {margin:0;}

这样的话，就不需要它的reset.css了，直接加在p的样式里面就好了。

精简版的reset.css

body, dl, dd, h1, h2, h3, h4, h5, h6, p, form{margin:0;}   
ol,ul{margin:0; padding:0;}

/*清除浮动*/
.clearfloat {
zoom:1;
}
.clearfloat:after {
    display: block;
    clear: both;
    content: "";
    visibility: hidden;
    height: 0;
}
/*a标签reset样式*/
a {
    display: block;
    color: #000;
    text-decoration: none;
}
/*li标签reset样式*/
li {
    list-style-type: none;
}
